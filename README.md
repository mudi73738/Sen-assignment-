PROJECT TITLE: Student Attendance Management System

1. REQUIREMENT ANALYSIS
The system is designed to help lecturers manage student attendance efficiently.
Functional Requirements:
- Add students
- Mark attendance
- Store attendance records
- View attendance history

Non-Functional Requirements:
- Easy to use
- Reliable
- Fast execution

2. SYSTEM DESIGN
The system is a console-based application.
Design includes:
- Menu-driven interface
- Student data stored in memory
- Attendance stored in a text file

3. IMPLEMENTATION
The system is implemented using Python.
Major components:
- add_student()
- mark_attendance()
- view_attendance()
- menu()

4. TESTING
Testing involved:
- Adding multiple students
- Marking attendance
- Viewing stored records

5. DEPLOYMENT
The application can be deployed by running:
python attendance.py

Source code can be uploaded to GitHub.

6. MAINTENANCE
Future improvements:
- Add date and time
- Convert to GUI
- Use database storage
